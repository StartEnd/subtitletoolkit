Most web players prefer `VTT`, even if your subtitles started as SRT or ASS.

## Typical starting points

- You downloaded subtitles as `SRT`
- Your editing workflow produced `ASS`
- Your archive or old project still uses `SRT`

## Typical destination

The destination is usually `VTT`, because that is the most browser-friendly format for HTML5 video workflows. If you are deciding whether that is still true for your player, see [Best subtitle format for HTML5 video](/guides/best-subtitle-format-for-html5-video).

## Conversion rules

- `SRT -> VTT` when you just need browser playback
- `ASS -> VTT` when you are leaving a styled editing workflow
- `VTT -> SRT` when a collaborator or tool still needs the simpler format

For the most common browser conversion path, [How to convert SRT to VTT for HTML5 video](/guides/how-to-convert-srt-to-vtt-for-html5-video) is the direct step-by-step page.

## Before you ship the file

Check these points:

- Timing still looks correct
- Multi-line captions still break where expected
- Browser playback accepts the file without errors

If browser playback still rejects the file, [How to fix invalid WebVTT timestamps](/guides/how-to-fix-invalid-webvtt-timestamps) is usually the next debug step.
