For HTML5 video, `VTT` is usually the safest default.

## Why VTT works best

- It is designed for web playback.
- Browser players understand it directly.
- It supports the `WEBVTT` structure expected by HTML5 environments.

## Why SRT still shows up

SRT is still common because many subtitle tools export it first. That means web workflows often need a conversion step before final playback. If that is your case, go straight to [How to convert SRT to VTT for HTML5 video](/guides/how-to-convert-srt-to-vtt-for-html5-video/).

## Where ASS fits

ASS is useful while editing, but it is usually not the final browser delivery format. If your source subtitles are ASS, converting to VTT is often the cleanest web step. The broader workflow is covered in [How to convert subtitle files for web players](/guides/how-to-convert-subtitle-files-for-web-players/).

## Practical workflow

1. Keep editing subtitles in the format your workflow prefers.
2. Convert the final browser version to `VTT`.
3. Test the VTT output in the target player.

If the file still fails after conversion, check [How to fix invalid WebVTT timestamps](/guides/how-to-fix-invalid-webvtt-timestamps/) before debugging the player. For a broader troubleshooting walkthrough, see [Why subtitles do not show in HTML5 video](/guides/why-subtitles-do-not-show-in-html5-video/).
