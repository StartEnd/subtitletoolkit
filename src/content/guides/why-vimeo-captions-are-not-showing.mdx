Vimeo caption problems often come from the uploaded caption file or the way the language track is configured.

## Quick answer

Use a clean WebVTT file for Vimeo caption workflows. If your source file is SRT or ASS, convert it with the [Vimeo Subtitle Converter](/tools/vimeo-subtitle-converter/) before uploading or testing embed captions.

Then check that the caption language track is enabled and available in the player.

## What to check

Check these items:

- the caption file is valid WebVTT
- the file has a `WEBVTT` header
- timestamps use dots, not commas
- the correct language is assigned
- captions are enabled in the Vimeo player
- the embed settings allow captions to appear

## Step-by-step workflow

1. Convert the subtitle file to VTT if needed.
2. Validate the VTT output.
3. Upload or attach the caption file in Vimeo.
4. Confirm the language track settings.
5. Test the video on the Vimeo page.
6. Test the embedded player if the video is used on your site.

## Common mistakes

### Uploading a file that looks like VTT but is really SRT

VTT needs a header and dot-based timestamps. Convert the file properly.

### Forgetting the embed

Captions can behave differently between the Vimeo page and an embedded player. Test the context where viewers watch.

### Replacing the source file

Keep the original subtitle source. Use VTT as the delivery copy for Vimeo.
